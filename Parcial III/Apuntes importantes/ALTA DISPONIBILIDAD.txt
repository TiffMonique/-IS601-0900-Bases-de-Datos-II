-----ALTA DISPONIBILIDAD-----
LA IDEA ES QUE LOS USUARIO TENGA DISPONIBILIDAD DE DATOS SIEMPRE, PARA PODER LOGRAR ESO. DEBEMOS TENER EN CUENTA 
LA CANTIDAD DE TRANSACCIONES QUE SE VAN A REALIZAR A DIARIO Y TENER UNA BASE DE DATOS CONFIGURADA QUE PERMITA ALMACENAR 
TODOS LOS DATOS QUE SON DE IMPORTANCIA PARA EL NEGOCIO.

TODA ESA INFORMACION ES IMPORTANTE QUE ESTE DISPONIBLE.
DISPONIBILIDAD: TIEMPO EN QUE LOS DATOS ESTAN DISPONIBLES.
SE DEFINE QUE TODO LOS COMPONENTES HW Y SW ESTEN DISPONIBLES PARA LOS USUARIOS. CONSTITUYEN LA ALTA DISPONBLIDAD.
EL TIEMPO EN QUE LOS DATOS ESTARAN DISPONIBLES TOMANDO EN CUENTA LOS RECURSOS DE LA EMPRESA

NO TODAS LAS EMPRESAS NECESITAN SOLUCION DE ESTE TIPO. DEPENDERA DE QUE TAN NECESARIO ES QUE LOS DATOS ESTEN DISPONIBLES PARA LOS USUARIOS.
MINIMO SE NECESITAN DOS SERVIDORES. PARA IMPLEMENTAR LA ALTA DISPONIBILIDAD. TENER EN CUENTA EL GESTOR.


-----ALTA DISPONIBILIDAD EN SQL-------
DATABASE MIRRORING : ES UNA BD REFLEJADA EN OTRA. ESTA CONFIGURACION SE HACE Y EL GESTOR SE ENCARGA DE GUARDAR TODAS LAS ACCIONES QUE SE HAGAN EN LA BASE DE DATOS REFLEJADA

--CONMUTACION POR ERROR CASI INMEDIATA.
---NECESITAMOS UNA TERCER INSTANCIA O SEA UN SERVIDOR COMO TESTIGO ESTARA REALIZANDO COMUNICACION CON LOSDOS SERVIDORES.REALIZA LA TAREA DE PASAR DE PASIVO A ACTIVO.
SE HACE SIN PERDIDA DE INFORMACION.


--CONMUTACION MANUAL POR ERROR. 
NO TENEMOS UN TESTIGO. SI EL ENLACE DE COMUNICACION SE CAE , DEBO ESTAR PENDIENTE.Y CAMBIAR LOS ROLES DE LOS SERVIDORES DE ACTIVO A PASIVO. 
LAS BASES DE DATOS DEBEN ESTAR SINCRONIZADAS.

--CONMUTACION POR ERROR.
PUEDE HABER PERDIDA DE INFORMACION. DEBEMOS BUSCAR UN MECANISMO PARA RECUPERAR LA INFORMACION

--CLUSTERING
SI EL SERVIDOR SE CAE , ES TRANSPARENTE. NO HAY QUE LEVANTAR SERVIDORES. LOS DEMAS SERVIDORES SIGUEN FUNCIONANDO. LA CONMUTACION AUTOMATICA POR ERROR SE REALIZAEN CASO
DE ERRORES DE HW, ERRORES DEL SO , ERRORES DE APPLICACION. LA INVERSION PUEDE SER UN POCO MAYOR


----REPLICACION
ES SIMILAR AL DATA BASE MIRRORING, LA DIFERENCIA ES QUE NO TENEMOS TESTIGO. AL MENOS TENDREMOS DOS SERVIDORES. Y TENDREMOS UNA BD PRINCIPAL Y LUEGO SE PASAN A LA O LAS BD/S SECUNDARIAS
-PARA PODER CONFIGURARLA NECESITO VER QUE SE EST
HAY VARIAS OPCIONES DE REPLICACION.
EN LA REPLICACION NO PUEDO AGREGAR EL TESTIGO. TENGO QUE ESTAR PENDIENTE. GARANTIZAR QUE LOS DATOS ESTEN SIEMPRE DISPONIBLES.
EN NINGUN MOMENTO TENEMOS QUE HACER BACKUP.
